\chapter*{General Introduction}
\addcontentsline{toc}{chapter}{General Introduction}
\label{ch:introduction}

Compiler optimization is a central concern in modern computing. The performance of a program depends not only on the algorithms it implements but also on how the compiler transforms high-level code into efficient machine instructions. Compilers apply sequences of transformations—such as loop tiling, parallelization, fusion, interchange, and vectorization—to improve execution time and resource utilization. However, determining the optimal sequence and parameters for these transformations is a difficult problem for two reasons. First, the \textbf{phase ordering problem}: transformations interact in non-linear ways, and a locally beneficial decision (such as fusing two operations) can preclude a globally superior configuration (such as parallelizing a different operation). Second, \textbf{hardware diversity}: the optimal transformation strategy for a 64-core server with large caches differs fundamentally from the strategy for a low-power embedded processor with limited SIMD capabilities. Traditional compilers address these challenges with hand-crafted heuristics, which are expensive to develop, fragile across architectures, and unable to adapt to new program patterns.

The MLIR compiler infrastructure, introduced by Lattner et al.~\cite{lattner2021mlir}, provides a unified intermediate representation designed to lower the barrier for compiler development and optimization. MLIR represents programs at multiple levels of abstraction through \textit{dialects}, and its \textit{transform dialect} enables the systematic application and composition of code transformations. This structured representation makes MLIR an attractive target for automated optimization techniques, including those based on machine learning.

Reinforcement learning (RL) offers a data-driven alternative to hand-crafted heuristics. In the RL framework, an agent interacts with an environment by observing its current state, selecting an action, and receiving a numerical reward that indicates the quality of that action. Through repeated interaction, the agent learns a \textit{policy}—a mapping from states to actions—that maximizes cumulative reward over time. Proximal Policy Optimization (PPO), introduced by Schulman et al.~\cite{schulman2017ppo}, is a policy gradient algorithm that has been successfully applied to domains ranging from game playing to robotics, and more recently to compiler optimization.

The application of RL to compiler optimization in MLIR was pioneered by Bendib et al.~\cite{bendib2024reinforcement}, who developed a reinforcement learning environment that enables an agent to optimize individual Linalg operations by selecting and parameterizing loop transformations. Building on this foundation, the previous team in this project~\cite{PREVIOUS_TEAM} extended the agent to optimize full neural network computation graphs. Their system introduced an abstract syntax tree (AST) based state representation with a recursive Long Short-Term Memory (LSTM) encoder, a fusion action that merges producer-consumer operation pairs, and a dataset composed of synthesized operation sequences as well as blocks extracted from convolutional neural network models. Their evaluation on VGG, ResNet, and MobileNet showed that the RL agent could achieve competitive speedups—notably a $2.5\times$ improvement on MobileNetV2 compared to PyTorch—but underperformed on architectures with highly regular patterns (VGG, $1.09\times$) or when the search space was constrained.

The analysis by the previous team identified several limitations of their system that motivate the present work. First, the agent's observation did not include any information about the target hardware, forcing it to learn hardware-appropriate strategies through trial and error rather than conditioning decisions on known platform characteristics. Second, the reward function was entirely sparse: the agent received feedback only at the end of an episode (a log-speedup reward), which slows learning and provides no guidance on intermediate transformation choices. Third, the recursive LSTM encoder, while effective for shallow sequences, can struggle to capture long-range dependencies in deeply nested loops where the relationship between outer and inner loop levels is critical. Fourth, the action space was limited to six transformation types, excluding operations such as padding and loop unrolling that can improve memory alignment and instruction-level parallelism.

This thesis addresses these four limitations through a set of complementary improvements to the RL-based auto-scheduler, culminating in a unified system (V4) that integrates them. Specifically, we develop:

\begin{enumerate}
    \item A \textbf{hardware-aware observation space} (V1) that encodes the target processor's cache hierarchy, core count, SIMD width, and clock frequency directly into the agent's input, enabling platform-specific optimization decisions.
    \item A \textbf{shaped reward function} (V2) that supplements the terminal speedup reward with dense intermediate rewards computed from static code analysis—including arithmetic intensity, vectorizability, and parallelization potential—providing learning signal at every transformation step.
    \item A \textbf{transformer-based loop nest encoder} (V3) that replaces the recursive LSTM with structured tokenization and multi-head self-attention, allowing the model to directly attend to relationships between any pair of loop levels in the nest.
    \item An \textbf{expanded action space} that introduces pad, pack, and unroll transformations, giving the agent finer control over data layout and instruction-level parallelism.
\end{enumerate}

These improvements are developed and evaluated incrementally (versions V1 through V3), then combined into version V4, which integrates hardware awareness, shaped rewards, and the transformer encoder. A comparative evaluation framework, including a Streamlit dashboard, is provided to analyze performance differences across versions.

\textbf{Thesis organization.} The remainder of this manuscript is structured as follows. Chapter~1 introduces the fundamentals of compiler code optimization: the compilation pipeline, loop nest transformations, and the MLIR infrastructure. Chapter~2 provides background on reinforcement learning, from Markov decision processes through policy gradient methods and PPO. Chapter~3 surveys related work on applying RL to compiler optimization, including Halide, Tiramisu, CompilerGym, and PolyGym, and concludes with a comparative synthesis. Chapter~4 presents the design and implementation of our system, detailing the observation space, action space, reward function, and policy architecture for each version. Chapter~5 describes the experimental methodology and results. A general conclusion summarizes our findings and outlines directions for future research.
